AI Integration (Optional)

Connect Millrect to AI assistants such as Claude Desktop to operate the app remotely. This is not required for normal drawing — use it when you want AI help creating drawings.

Not required for everyday use

Millrect works fully without AI. This page is reference material for users who want to set up AI integration.

What is AI integration?

While Millrect is running, an AI (such as Claude) can add shapes, move objects, regenerate 3D, and more on your behalf. The AI gives instructions; Millrect performs the actual geometry and rendering.

Example: you can tell the AI things like “Draw a 100 mm wide rectangle” or “Add a top view page,” and it will relay those requests to Millrect.

Requirements

  • Millrect must be running (AI integration is only active while the app is open)
  • An AI client that supports MCP (Model Context Protocol), such as Claude Desktop
  • MCP server configuration (bundled with the desktop app; requires Node.js on PATH)

MCP server setup

The desktop build bundles an MCP server. While Millrect is running, point your AI client at:

EnvironmentExample path to server.js
macOS (.app) /Applications/Millrect.app/Contents/Resources/mcp/server.js
Windows C:\Program Files\Millrect\resources\mcp\server.js (depends on install dir)
Development (repo) mcp/server.js

Claude Desktop (claude_desktop_config.json) example:

{
  "mcpServers": {
    "millrect": {
      "command": "node",
      "args": ["/Applications/Millrect.app/Contents/Resources/mcp/server.js"]
    }
  }
}

On Windows, use the path from the table above and escape backslashes for JSON. Launch Millrect first, then restart Claude Desktop.

Other MCP clients (e.g. Cursor) use the same node + server.js entry. See the Developer guide (MCP) and AGENT.md in the repository for agent instructions.

Basic workflow with AI

  1. Launch Millrect
    Open the app as usual and create or open a project.
  2. Connect the AI client to Millrect
    Once MCP is configured, the AI can read Millrect’s state.
  3. Give instructions to the AI
    Ask in natural language, e.g. “Show me the current drawing” or “Draw a 24×100 joint outline.”
  4. Review results in the app
    Changes made by the AI appear on the Millrect canvas. Edit manually if needed.

Useful levels of AI work

Millrect’s AI integration is centered on structured drawing, part, and judgment APIs rather than UI clicking. The usual flow is to inspect the current document, generate drawings from millimeter specs, then validate 3D readiness and manufacturing constraints.

LayerMain toolsBest for
Contextget_project_context, validate_3d_readinessRead pages, face directions, profiles, and 3D readiness
Intent APIcreate_multiview_box, layout_rect_mmGenerate drawings from width/depth/height in mm
Part DSLcompile_part_dsl, apply_part_dsl, update_part_paramCreate boxes, panels, L-brackets, enclosures, holes, slots, and fillets declaratively
Manufacturing checkvalidate_manufacturabilityCheck hole size, wall thickness, fillet, and other rule warnings
Reviewcapture_screenshot, append_artifact_logRecord visual review and feed the next revision

Sketch digitize (reference image + ghosts)

Millrect does not run Vision internally. An external AI (e.g. Claude) analyzes the image and sends mm-based shape proposals via MCP. You review proposals in the app before confirming them.

  1. Place reference image
    MCP load_reference_image, or UI: Pages → Reference image → Load image.
  2. Move & resize (optional)
    UI: Move & resize → adjust on canvas → click the same button again to exit. MCP can also set x / y / widthMm on load_reference_image.
  3. Calibrate scale
    set_reference_scale_anchor (both ends of a known-length line + mm), or UI Calibrate scale.
  4. Place proposals
    AI calls digitize_sketch with rect/circle/line proposals (mm) → shown as ghosts.
  5. Confirm and edit
    UI or confirm_digitize_proposals to finalize; adjust dimensions manually if needed.
Ghosts are not included in 3D

Ghost shapes are excluded from Profile extraction and 3D until confirmed. Always confirm before generating solids.

See the Developer guide (API) and AGENT.md (sketch digitization section) for the full agent API.

Notes

  • To build 3D, you still need multiple orthographic view pages, same as manual workflow
  • Shapes drawn by AI can always be edited or deleted by hand
  • AI integration is a helper — always verify the final design yourself
