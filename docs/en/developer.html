Development setup

After cloning the repository, install dependencies with Node.js and launch the app.

npm install
npm run fonts:fetch   # Gen Interface JP (first run / updates)
npm run dev           # Electron development
npm run build         # Package build (dist/)

Bundled fonts (Gen Interface JP) may not be in the Git tree. Run npm run fonts:fetch before text rendering or outline conversion works. The app UI, docs, and landing page use the same typeface (app/fonts/, docs/fonts/, site/fonts/).

You can serve app/index.html with a static server to try the UI in a browser, but the MCP WebSocket, text outline, and file dialogs require the Electron build.

Desktop release procedure (version bump, build, GitHub Release): docs/RELEASE.md (Japanese: docs/RELEASE.ja.md).

E2E tests: npm run test:e2e (Playwright). MCP server alone: node mcp/server.js while Millrect is running.

No bundler — files load via <script src> into the global scope. When adding files, respect the order in app/index.html.

Boundary rule: runnable products belong under apps/; reusable libraries belong under packages/. The existing 2D editor is still in app/ for compatibility with Electron, build packaging, docs, and tests. When it is moved, the target should be apps/millrect-editor, not packages/. Only reusable 2D primitives such as drawing schema, profile extraction, command reducers, or rendering helpers should be split into packages.

Architecture

Design principles

PrincipleMeaning
Edit 2D drawings only3D is derived; always regeneratable via update3DScene()
Undo/Redo protectionpushHistory() records document parts only; zoom/pan/tool are excluded
Stable shape IDsDo not change shape IDs; use genId() on collision
Dimensions separate from shapesaddShape({type:"dimension",...}) routes to page.dimensions[]
3D from drawingsEach page’s viewDefinition + contours; per-shape feature.depth is removed

Data flow (3D generation)

pages[] (viewDefinition.type + contour shapes per page)
  ↓ shapeToProfile() / extractProfilesFromPage()  ← profiles.js
Profile (derived; not stored in State)
  ↓ _profileToThreeShapesForView() + ExtrudeGeometry (internal sweep)
  ↓ CSG union (within view) → CSG intersect (between axis pairs)
Mesh → STL

At least two orthogonal views are required (e.g. top + front). One page alone produces no solid.

Separated 3D pipeline

The drawing document remains the source of truth. New 3D accuracy work should be isolated in the pipeline packages instead of changing the 2D editor UI directly. 

Project JSON / 2D drawings
  ↓ packages/schema
schema validation
  ↓ packages/model-generator
Model IR (editable operations / features, not only mesh)
  ↓ packages/geometry-core
geometry data / mesh / export-ready data
  ↓ packages/model-viewer
Three.js display

app/js/3d-view.js still owns the legacy multiview CSG preview and STL path for compatibility. It also refreshes the new wrapper state, available after update3DScene() via get3DModelPipelineState().

State separation

_state = {
  // Document (Undo / export)
  projectName, unit, fonts, pages, partIntent, projectBrief,

  // UI (not Undo)
  zoom, panX, panY, activeTool,
  currentPageId, currentLayerId,
  selectedShapeIds, showGrid, gridSize, snapEnabled,
}

Coordinates

ConceptValue
Unitreal units (1 mm = 10 units)
A4 landscape2970 × 2100
A4 portrait2100 × 2970
A3 landscape4200 × 2970

With scale 1:10 (scale: { numerator:1, denominator:10 }):

paperUnit = realUnit × numerator / denominator
realUnit  = paperUnit × denominator / numerator

Data schemas

State

See _state under Architecture. pages[] is the core of the document.

ProjectFont (state.fonts[])

{
  id: string,
  family: string,           // e.g. "Roboto"
  cssUrl: string,           // Google Fonts CSS (display)
  fileUrl: string,          // TTF URL (400 / Regular)
  fileUrlBold?: string,     // TTF URL (700 / Bold)
  source: "google",
  libraryId?: string        // optional link to user library entry
}

Page

{
  id, name,
  paper: 'A4'|'A3'|'A2'|'A1',
  orientation: 'landscape'|'portrait',
  scale: { numerator, denominator },
  viewDefinition: {
    type: 'top'|'bottom'|'front'|'back'|'right'|'left'|'section'|'detail'|null,
    normal: [x,y,z]|null,
    up: [x,y,z]|null,
  },
  dimensions: [Dimension],   // separate from shapes[]
  referenceImage?: { dataUrl, x, y, width, height, opacity },
  layers: [Layer],
  constraints?: [Constraint],
}

Layer

{ id, name, visible, locked, shapes: [Shape] }

Shape

// Common
{ id, type, stroke, fill, strokeWidth: 'thin'|'medium'|'thick',
  rotation?, flipH?, flipV?,   // reflected in SVG, BBox, Profile, and 3D
  ghost?: boolean }             // unconfirmed digitize proposal (excluded from 3D)

// Types
rect:    { x, y, width, height, rx? }
circle:  { cx, cy, r }
line:    { x1, y1, x2, y2 }
text:    { x, y, text, fontSize?, fontFamily?, fontWeight?, textAlign?, lineHeight?, width? }
         // text color uses stroke (not fill)
bezier:  { nodes: [{x, y, h1:{x,y}|null, h2:{x,y}|null}], closed: bool }
path:    { contours: ring[][][] }   // polygon-clipping format
group:   { children: Shape[] }

Dimension (page.dimensions[])

{
  id, type: 'dimension',
  dimensionType: 'horizontal'|'vertical',
  from: {x, y}, to: {x, y},
  offset: number,
  color?, lineWidth?, textSize?, arrowStyle?, fontFamily?,
  value?, decimals?, prefix?, suffix?,
  textOffsetX?, textOffsetY?, textRotation?,
}

addShape({ type:'dimension', ... }) is automatically routed to page.dimensions[].

Profile (derived; not stored in State)

{
  id: 'profile-{shapeId}',
  sourceId: string,
  pageId: string,
  rings: [[[x,y]...]],   // [0]=outer, [1..]=holes
  bbox: { x, y, w, h },
  area: number,
}

Supported: rect / circle / bezier(closed) / path. group is recursively expanded into child shapes for Profile extraction. Not supported: line / text / dimension / bezier(open).

findShapeById(id) → { shape, layer, page, isDimension } (layer is null when isDimension === true).

Taste Memory (design)

Extension spec for recording user aesthetic judgments in three layers (Global / Project / Artifact). The Project layer (state.projectBrief), Global layer (IndexedDB tasteGlobal), and Artifact layer (projectBrief.artifactLog) are implemented. Separate from geometric partIntent. The human-facing UI lives in the right panel under Pages tab → Taste brief.

  • docs/TASTE-MEMORY.md — schemas, workflow, MCP/API, promotion rules
  • docs/adr/0001-taste-memory-three-layer-model.md — architecture decision summary
  • get_taste_context / update_project_brief / record_decision / append_artifact_log — available via MCP
  • captures/ ZIP export is not implemented yet (project JSON stores relative paths only)

MCP / WebSocket

While Millrect is running, main.js starts a WebSocket server at ws://127.0.0.1:23450 (or port+1 if busy). The auth token is written to ~/.millrect/millrect-ws-token. mcp/server.js bridges stdio MCP ↔ WebSocket.

Basic workflow

  1. Launch Millrect — WS server and token file are created
  2. Connect MCP client — register node mcp/server.js
  3. get_project_context — check new vs in-progress, profile count, viewDefinition
  4. Prefer Intent API — create_multiview_box, validate_3d_readiness, etc.
  5. Fine edits — apply_commands for individual shapes
  6. Verify 3D — update_3d_scene → get_3d_scene_status

Electron

The desktop app loads app/index.html in Electron.

Security

  • contextIsolation: true — isolate renderer from Node
  • nodeIntegration: false
  • sandbox: true
  • Renderer uses only window.electronAPI via preload.js
  • WebSocket MCP binds to 127.0.0.1 only; token auth required
  • App CSP: Content-Security-Policy meta tag in app/index.html

Known patterns & gotchas

  • Bezier overlay: always render() then renderBezierOverlay()
  • Bezier mouse handler: check before if (!_ds) return
  • Edit mode: _bezierEditId works regardless of active tool
  • rotation / flipH / flipV: reflected in SVG, getShapeBBox, alignment, Profile, and 3D. Transform order is rotate → flip
  • Ghost shapes: shape.ghost excluded from Profile / 3D until confirmed
  • Web Components: do not use display: contents in js/components/ (breaks absolute positioning)
  • Autosave: pushHistory() → onStateChanged() → 2s debounce → doAutosave()
  • shiftShape(): does not call pushHistory(); record history manually after direct edits
  • render(): RAF-debounced — rapid calls coalesce to one frame
  • 3D generation: do not use per-shape feature.depth; multiview CSG only
