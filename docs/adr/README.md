# Architecture Decision Records (ADR)

Millrect の重要な設計判断を短い ADR として残す。実装の詳細は各トピックの設計ドキュメント（例: [TASTE-MEMORY.md](../TASTE-MEMORY.md)）を参照。

| ADR | タイトル | 状態 |
|-----|----------|------|
| [0001](0001-taste-memory-three-layer-model.md) | Taste Memory — 3 層メモリモデル | Accepted |

## 書き方

- 1 ADR = 1 つの判断。ファイル名は `NNNN-kebab-title.md`
- ステータス: `Proposed` → `Accepted` → `Deprecated` / `Superseded`
- 実装が `Accepted` に達したら ADR を更新する
