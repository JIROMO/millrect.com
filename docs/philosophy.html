設計思想

Millrect は、図面を中心にした軽量な作図アプリです。3D、AI、出力機能は、図面づくりを置き換えるものではなく、図面から自然に広がる補助機能として設計しています。

  • 3D は派生物
  • 小さく、確かめやすく

  • 手元に残せる形式

図面が本体

Millrect で最初に触るものは 3D モデルではなく、上面図・正面図などの 2D 図面です。寸法、形状、ページ構成を図面として持つことで、後から見返したときに「何を作ろうとしていたか」が残ります。

3D 表示や STL 出力は便利ですが、編集の中心は常に図面です。図面を直せば、そこから立体を作り直せます。

3D は派生物

Millrect の 3D は、上面図・正面図など複数の投影図から生成されます。3D だけを直接こねるのではなく、根拠になる 2D の輪郭を残すことで、作業の説明性とやり直しやすさを優先しています。

考え方Millrect での扱い
編集対象2D 図面、ページ、寸法、輪郭
生成対象図面から作られる 3D プレビューと STL
やり直し図面を修正して再生成

小さく、確かめやすく

ブラウザでもデスクトップでも同じ図面データを扱えるようにし、作図、プレビュー、保存がひとつの流れでつながることを大事にしています。

機能を増やすときも、作図中の視線を奪わないこと、プロジェクトファイルとして読み書きしやすいこと、Undo / Redo の予測が立つことを優先します。

AI は操作者ではなく補助者

MCP 連携やスケッチ取り込みは、作図の入口を用意するための機能です。AI が勝手に確定するのではなく、提案を見て、必要なら調整してから図面に反映する流れを基本にしています。

確認できる自動化

AI 連携は「図面の代わり」ではなく、図面を作るための補助線です。最終的な形はキャンバス上で確認できる状態に残します。

手元に残せる形式

プロジェクトは JSON として保存でき、出力は PDF / SVG / STL など用途に合わせた形式を選べます。作ったものを特定の画面の中だけに閉じ込めず、後から開ける・渡せる・加工できる状態にすることを重視しています。
