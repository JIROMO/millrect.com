開発環境

リポジトリを clone したあと、Node.js で依存関係をインストールして起動します。

npm install
npm run fonts:fetch   # Gen Interface JP（初回・更新時）
npm run dev           # Electron 開発起動
npm run build         # 配布物ビルド（dist/）

同梱フォント（Gen Interface JP）は Git に含まれない場合があります。 テキスト表示・アウトライン化の前に npm run fonts:fetch を実行してください。 アプリ UI・ドキュメント・ランディングページも同じ書体を使います（app/fonts/ · docs/fonts/ · site/fonts/）。

ブラウザのみで UI を試す場合は app/index.html を静的サーバーで配信しても動きますが、 MCP WebSocket・テキストアウトライン・ファイルダイアログは Electron 版の機能です。

デスクトップ版のリリース手順（バージョン bump、ビルド、GitHub Release）は docs/RELEASE.ja.md（英語: docs/RELEASE.md）を参照してください。

E2E テスト: npm run test:e2e（Playwright）。 MCP サーバー単体: Millrect 起動中に node mcp/server.js。

バンドラなし — <script src> タグでグローバルスコープに読み込みます。 新しいファイルを追加するときは app/index.html の順序を守ってください。

境界ルール: 実行アプリは apps/、再利用可能なライブラリは packages/ に置きます。既存の 2D editor は Electron、build packaging、 docs、tests との互換性のため、現時点では app/ に残しています。 移動する場合の移行先は packages/ ではなく apps/millrect-editor です。packages/ に切り出すのは、 drawing schema、profile 抽出、command reducer、rendering helper など再利用可能な 2D primitive に限定します。

アーキテクチャ

設計原則

原則内容
編集するのは 2D 図面だけ3D は派生データ。いつでも update3DScene() で再生成可能
Undo/Redo の保護pushHistory() はドキュメント部分のみ。zoom/pan/tool は Undo 対象外
図形 ID の安定性shape ID は変更しない。重複時は genId() で再生成
dimension は shapes に混ぜないaddShape({type:"dimension",...}) → 自動で page.dimensions[] へ
3D は図面から導出各ページの viewDefinition + 輪郭。per-shape feature.depth は廃止

データフロー（3D 生成）

pages[]（各ページに viewDefinition.type + 輪郭図形）
  ↓ shapeToProfile() / extractProfilesFromPage()  ← profiles.js
Profile（State に保存しない派生データ）
  ↓ _profileToThreeShapesForView() + ExtrudeGeometry（内部スイープ）
  ↓ CSG union（同一ビュー内）→ CSG intersect（軸ペア間）
Mesh → STL

最低 2 軸のビューが必要（例: 上面図 + 正面図）。1 ページだけでは立体は生成されません。

分離した 3D パイプライン

図面ドキュメントを正とし、3D 精度改善は 2D 編集 UI ではなく pipeline package 内に閉じる方針です。 

Project JSON / 2D 図面
  ↓ packages/schema
schema validation
  ↓ packages/model-generator
Model IR（mesh だけではなく、編集可能な operation / feature）
  ↓ packages/geometry-core
geometry data / mesh / export-ready data
  ↓ packages/model-viewer
Three.js 表示

互換性のため、既存の多ビュー CSG preview / STL はまだ app/js/3d-view.js が担当します。 同時に新 wrapper state を更新しており、update3DScene() 後に get3DModelPipelineState() で確認できます。

State の分離

_state = {
  // ドキュメント（Undo 対象・エクスポート対象）
  projectName, unit, fonts, pages, partIntent, projectBrief,

  // UI（Undo 対象外）
  zoom, panX, panY, activeTool,
  currentPageId, currentLayerId,
  selectedShapeIds, showGrid, gridSize, snapEnabled,
}

pushHistory() / undo() / redo() は DOC_KEYS のみ操作します。

座標系

概念値
単位real units（1 mm = 10 units）
A4 横2970 × 2100
A4 縦2100 × 2970
A3 横4200 × 2970

縮尺 1:10（scale: { numerator:1, denominator:10 }）の換算:

paperUnit = realUnit × numerator / denominator
realUnit  = paperUnit × denominator / numerator

データスキーマ

State

上記 アーキテクチャ の _state を参照。pages[] がドキュメントの中心です。

ProjectFont（state.fonts[]）

{
  id: string,
  family: string,           // 例: "Roboto"
  cssUrl: string,           // Google Fonts CSS（表示用）
  fileUrl: string,          // TTF URL（400 / Regular）
  fileUrlBold?: string,     // TTF URL（700 / Bold）
  source: "google",
  libraryId?: string        // ユーザーライブラリとの対応（任意）
}

Page

{
  id, name,
  paper: 'A4'|'A3'|'A2'|'A1',
  orientation: 'landscape'|'portrait',
  scale: { numerator, denominator },
  viewDefinition: {
    type: 'top'|'bottom'|'front'|'back'|'right'|'left'|'section'|'detail'|null,
    normal: [x,y,z]|null,
    up: [x,y,z]|null,
  },
  dimensions: [Dimension],   // shapes[] とは独立
  referenceImage?: { dataUrl, x, y, width, height, opacity },
  layers: [Layer],
  constraints?: [Constraint],
}

Layer

{ id, name, visible, locked, shapes: [Shape] }

Shape

// 共通
{ id, type, stroke, fill, strokeWidth: 'thin'|'medium'|'thick',
  rotation?, flipH?, flipV?,   // SVG 表示・BBox・Profile・3D に反映
  ghost?: boolean }             // 確定前 digitize 提案（3D 非対象）

// 種類
rect:    { x, y, width, height, rx? }
circle:  { cx, cy, r }
line:    { x1, y1, x2, y2 }
text:    { x, y, text, fontSize?, fontFamily?, fontWeight?, textAlign?, lineHeight?, width? }
         // 文字色は stroke を使用（fill は使わない）
bezier:  { nodes: [{x, y, h1:{x,y}|null, h2:{x,y}|null}], closed: bool }
path:    { contours: ring[][][] }   // polygon-clipping 形式
group:   { children: Shape[] }

Dimension（page.dimensions[]）

{
  id, type: 'dimension',
  dimensionType: 'horizontal'|'vertical',
  from: {x, y}, to: {x, y},
  offset: number,
  color?, lineWidth?, textSize?, arrowStyle?, fontFamily?,
  value?, decimals?, prefix?, suffix?,
  textOffsetX?, textOffsetY?, textRotation?,
}

addShape({ type:'dimension', ... }) は自動的に page.dimensions[] にルーティングされます。

Profile（派生データ・State 保存なし）

{
  id: 'profile-{shapeId}',
  sourceId: string,
  pageId: string,
  rings: [[[x,y]...]],   // [0]=外周, [1..]=ホール
  bbox: { x, y, w, h },
  area: number,
}

対応: rect / circle / bezier(closed) / path。group は子図形を再帰的に展開して Profile 化します。非対応: line / text / dimension / bezier(open)。

findShapeById(id) → { shape, layer, page, isDimension }（isDimension === true のとき layer は null）。

Taste Memory（設計）

ユーザーの美意識・判断を 3 層（Global / Project / Artifact）で記録する拡張仕様です。 Project 層（state.projectBrief）、Global 層（IndexedDB tasteGlobal）、 Artifact 層（projectBrief.artifactLog）は実装済みです。幾何の partIntent とは分離します。 人間向けの確認 UI は右パネルの Pages タブ → 制作メモ にあります。

  • docs/TASTE-MEMORY.md — スキーマ、ワークフロー、MCP/API、昇格ルール
  • docs/adr/0001-taste-memory-three-layer-model.md — アーキテクチャ判断の要約
  • get_taste_context / update_project_brief / record_decision / append_artifact_log — MCP から利用可能
  • captures/ を含む ZIP export は未実装（プロジェクト JSON には相対パスのみを保存）

MCP / WebSocket

Millrect 起動中、main.js が ws://127.0.0.1:23450（ポート使用中なら +1）で WebSocket サーバーを立てます。 認証トークンは ~/.millrect/millrect-ws-token に書き出されます。 mcp/server.js が stdio MCP ↔ WS を橋渡しします。

Electron

デスクトップ版は Electron で app/index.html を読み込みます。

セキュリティ

  • contextIsolation: true — レンダラーと Node を分離
  • nodeIntegration: false
  • sandbox: true
  • レンダラーは preload.js 経由の window.electronAPI のみ使用
  • WebSocket MCP は 127.0.0.1 のみ。トークン認証必須
  • アプリ CSP: app/index.html の Content-Security-Policy メタタグ

既知のパターンと注意点

  • ベジェオーバーレイ: 必ず render() → renderBezierOverlay() の順
  • ベジェマウスハンドラ: if (!_ds) return ガードより前にチェック
  • Edit モード: _bezierEditId チェックはツール非依存
  • rotation / flipH / flipV: SVG 表示、getShapeBBox、整列、Profile、3D 生成に反映。変換順は回転 → 反転
  • ゴースト図形: shape.ghost は Profile / 3D 対象外。確定後に 3D 化
  • Web Components: js/components/ 内で display: contents 禁止（absolute 子の containing block が狂う）
  • Autosave: pushHistory() → onStateChanged() → debounce 2 秒 → doAutosave()
  • shiftShape(): pushHistory() を呼ばない。直接変更後は手動で履歴記録
  • render(): RAF debounce — 連続呼び出しは 1 フレームにまとまる
  • 3D 生成: feature.depth 等の per-shape 3D 指示は使わない。多ビュー CSG のみ
