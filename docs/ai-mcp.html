AI 連携（オプション）

Claude Desktop などの AI アシスタントから Millrect を操作できる連携機能です。 通常の作図には不要で、AI に図面作成を手伝ってもらいたい場合に使います。

通常利用では不要です

Millrect は手動で問題なく使えます。このページは、AI 連携を設定したい方向けの参考情報です。

AI 連携とは

Millrect を起動している間、AI（Claude など）が図形の追加・移動・3D 生成などを 代わりに実行できます。AI は「指示を出す」役割で、実際の幾何計算や描画は Millrect が行います。

例: 「幅 100 mm の矩形を描いて」「上面図ページを追加して」といった自然言語の指示を、 AI 経由で Millrect に伝えられます。

必要なもの

  • Millrect アプリが起動していること（AI 連携はアプリ起動中のみ有効）
  • Claude Desktop など、MCP（Model Context Protocol）に対応した AI クライアント
  • MCP サーバーの設定（配布物に同梱。Node.js が PATH に必要）

MCP サーバーの設定

デスクトップ版のインストール先に MCP サーバーが同梱されています。 Millrect を起動した状態で、AI クライアントから次のスクリプトを指定します。

環境server.js のパス（例）
macOS（.app） /Applications/Millrect.app/Contents/Resources/mcp/server.js
Windows C:\Program Files\Millrect\resources\mcp\server.js（インストール先による）
開発（リポジトリ） mcp/server.js

Claude Desktop（claude_desktop_config.json）の例:

{
  "mcpServers": {
    "millrect": {
      "command": "node",
      "args": ["/Applications/Millrect.app/Contents/Resources/mcp/server.js"]
    }
  }
}

Windows ではパスを上表の server.js に合わせ、バックスラッシュを JSON 用にエスケープしてください。 設定後、Millrect を起動してから Claude Desktop を再起動します。

Cursor など他の MCP クライアントも同様に node + server.js を指定できます。 操作の詳細は 開発者ガイド（MCP） およびリポジトリの AGENT.ja.md を参照してください。

AI との基本的な流れ

  1. Millrect を起動
    通常どおりアプリを開き、プロジェクトを作成または開きます。
  2. AI クライアントで Millrect に接続
    MCP 設定が済んでいれば、AI から Millrect の状態を読み取れます。
  3. AI に指示する
    「現在の図面を教えて」「24×100 のジョイント外形を描いて」など、自然言語で依頼します。
  4. 結果をアプリで確認
    AI が操作した内容は、Millrect の画面上に反映されます。必要なら手動で修正してください。

AI に頼みやすい作業単位

Millrect の AI 連携は、画面クリックを自動化するよりも、図面・部品・判断を構造化して渡すための API を中心にしています。 まず状況を読み、必要なら Taste Memory の brief を確認し、mm 指定で図面を生成してから 3D と製造ルールを検証する流れが基本です。

レイヤー主なツール向いている依頼
状況確認get_project_context, validate_3d_readiness現在のページ、面の向き、3D 化できるかを確認
Intent APIcreate_multiview_box, layout_rect_mm幅・奥行き・高さなど mm 指定で素早く図面化
Part DSLcompile_part_dsl, apply_part_dsl, update_part_param箱、パネル、L 字ブラケット、筐体、穴・スロット・フィレットなどを宣言的に作る
製造チェックvalidate_manufacturability穴径、壁厚、フィレットなど製造ルール上の注意点を確認
レビューcapture_screenshot, append_artifact_log見た目・意図との差分を記録し、次の修正へつなげる

Taste Memory（判断の記録）

Taste Memory は、AI が「なぜその形にするのか」を参照できるようにするための記録レイヤーです。 会話全文ではなく、案件の意図、制約、参考、設計方針、採用・却下の理由、レビュー結果を構造化して保存します。 アプリ上では右パネルの Pages タブ → 制作メモ で状態を確認できます。

保存先内容用途
projectBrief案件ごとの intent / constraints / designPrinciples / decisions / artifactLogこのプロジェクトで AI が守る判断基準
Global Memory複数プロジェクトで繰り返し確認された好み新規プロジェクト開始時の初期 brief
Artifact Logキャプチャ、3D 状態、評価、採否レビュー後の修正理由を残す

関連ツール: get_taste_context, update_project_brief, record_decision, set_project_phase, append_session_learnings, list_global_principles, promote_principle, append_artifact_log。 詳細は 開発者ガイド — Taste Memory と docs/TASTE-MEMORY.md を参照してください。

スケッチ digitize（参照画像 + ゴースト）

Millrect 内に Vision（画像認識）はありません。Claude など外部 AI が画像を解析し、 MCP 経由で mm 指定の図形提案を渡す設計です。人間はアプリ上で提案を確認してから確定します。

  1. 参照画像を配置
    MCP load_reference_image、または UI の Pages → 参照画像 → 画像を読み込む。
  2. 位置・サイズ（任意）
    UI の 位置・サイズを編集 → キャンバスで調整 → 同ボタンを再クリックで終了。MCP では load_reference_image の x / y / widthMm 等でも指定可。
  3. スケール校正
    set_reference_scale_anchor（実寸がわかる線の 2 端 + mm）、または UI の スケール校正。
  4. 図形提案を配置
    AI が digitize_sketch で矩形・円・線の proposals（mm）を送る → ゴースト表示。
  5. 確定して編集
    UI または confirm_digitize_proposals で通常図形に。必要なら手動で寸法を直す。
ゴーストは 3D に含まれません

確定前のゴースト図形は 3D 生成・Profile 抽出の対象外です。立体化する前に必ず確定してください。

エージェント向けの詳細 API は 開発者ガイド（API） と AGENT.ja.md（スケッチ取り込み章）を参照してください。

注意点

  • 3D を作るには、手動操作と同様に複数の投影図ページが必要です
  • AI が描いた図形も、いつでも手動で編集・削除できます
  • AI 連携は補助機能です。最終的な設計内容の確認はご自身で行ってください
